- .idea 폴더란?
  - JetBrains IDE가 프로젝트의 환경 설정, 구조, 편집기 설정 등을 저장하기 위해 자동으로 생성하는 디렉토리이다.
  - 팀원 간 공유할 수도 있고, 개인 설정만 포함될 수도 있어서 .gitignore로 제외하기도 한다.
  
- .idea 폴더에 들어있는 파일들
  - .gitignore(중복될 수 있으므로 Git에 포함 X)
  
    - JetBrains IDE 내부에서 무시할 파일/디렉토리 목록을 지정하기 위한 .gitignore 파일이다.
      Git 자체의 .gitignore와는 별개이다. **뭐가 다른지 검색해보자**
  
      - Git 자체의 .gitignore
        - Git 커밋에서 제외할 파일/폴더 지정하는 Git 버전 관리 시스템
      - .idea/.gitignore
        - IDE가 내부에서 무시할 파일/폴더 지정하는 JetBrains IDE 내부에서 작동
      - JetBrains IDE(Integrated Development Environment)
        - JetBrains라는 회사가 만든 여러 가지 IDE들의 총칭
  
      - IntelliJ IDEA(Integrated Development Environment A)
        - JetBrains에서 만든 IDE 중 하나로 Java와 JVM 언어를 위한 IDE
  
  - compiler.xml(개인 설정이므로 Git에 포함 X)
  
    - 컴파일러 관련 설정을 저장한다.(예를 들어 자동 빌드 켜기/끄기, 프로젝트 전체에 대한 빌드 동작 설정 등)
  
  - gradle.xml(프로젝트를 공유할 때 Gradle 프로젝트로 자동 인식되도록 조와주므로 Git에 포함 O)
  
    - 해당 프로젝트가 Gradle 프로젝트임을 IDE에 알리고, Gradle 관련설정(예: Wrapper 사용 여부, 자동 import 등)을 포함한다.
  
  - jeonsaeyukjun-be.iml(다른 개발자도 동일한 프로젝트 구조로 열 수 있게 하기위해 Git에 포함 O)
  
    - .iml 파일은 프로젝트의 모듈 설정을 정의한다.(소스 /테스트 디렉토리 구조, 의존성 경로, 빌드 경로 등)
  
  - misc.xml(팀 전체가 동일한 JDK/언어 수준을 유지할 수 있게 하기 위해 Git에 포함 O)
  
    - 프로젝트 전역 설정을 저장한다.(사용 JDK 버전, 언어 수준, 프로젝트 포맷 버전)
  
  - modules.xml(어떤 모듈이 프로젝트에 포함되어 있는지 명확하게 알 수 있기에 Git에 포함 O)
  
    - 프로젝트에 포함된 모듈 리스트와 각 모듈의 .iml 파일 연결 정보를 저장한다.
  
  - uiDesigner.xml(GUI 디자이너를 사용하지 않으면 무시 가능하기에 Git에 포함 X)
  
    - JetBrains의 GUI 디자이너(폼 디자이너)를 사용할 때 설정을 저장한다.
  
  - vcs.xml(개인 개발 환경에 따라 설정이 다를 수 있으므로 Git에 포함 X)
  
    - VCS(Version Control System, 예: Git, SVN 등) 관련 설정을 저장한다.(Git 루트, 커밋 메시지 템플릿, 자동 커밋 옵션 등)
  
  - workspace.xml(충돌이 자주 나고, 다른 사람에게 의미가 없으므로 Git에 **절대** 포함 X)
  
    - 개인 작업 공간 상태 저장(열려 있는 탭, 디버깅 상태, 브레이크포인트, 커서 위치 등)
